Security

Enterprise-ready by design

Multi-tenant architecture with workspace isolation, team-based RBAC, encrypted secrets, and conversation isolation — so one organization never sees another’s data.

Multi-tenant isolation

Tenant boundaries at the database and vector store. No cross-organization data leakage.

End-user identity forwarding

Forward signed-in identity via identify() so actions run as the real user — passwords never touch Qefro.

Zero-trust style authorization

Business actions authorize with the end-user’s identity by default — not a shared organization secret.

Audit & execution logs

Conversation history and Business Tool runs stay attached for accountability and review.

Team-based RBAC

Owner, Admin, and Member roles control documents, actions, billing, and invites.

Secure action execution

Encrypted secrets, SSRF protections, schema validation, and workspace isolation.

Your auth stays yours

Qefro is not an identity provider. Forward signed-in identity with identify() so business actions run on behalf of real users — passwords never touch Qefro.

  id: user.id,
  email: user.email,
