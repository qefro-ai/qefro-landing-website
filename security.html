Security

Authentication stays in your app. Qefro never stores passwords. Secrets are encrypted, messages are bounded and scrubbed, and every tool call is logged.

Your auth stays yours

Qefro is not an identity provider. Forward JWTs or session ids via identify() — end-user passwords are never stored or processed by Qefro.

Widget JWT auth

Embeds load with short-lived widget tokens. Messages are bounded (8 KB), PII is scrubbed before model calls, and tenants cannot access each other’s data.

Tenant & workspace isolation

Multi-tenant by design at the database and vector store level. Public and private workspaces control which knowledge and tools each assistant can use.

Encrypted secrets & logs

Business Tool credentials encrypted at rest. HTTPS-only outbound calls with SSRF protections and DNS-pinned webhooks. Full conversation and tool execution history.

Launch-hardened platform controls

Built for production customer support — not demo chatbots.

Access control

  • Owner / Admin / Member RBAC
  • Email OTP — no password storage
  • Billing actions restricted to owners
  • Workspace-scoped documents & tools

Data handling

  • PII scrubbing on outbound LLM calls
  • Never used to train AI models
  • Encrypted at rest & in transit
  • Document deletion & reindex

Tool execution

  • OpenAPI schema validation
  • SSRF & DNS pinning for webhooks
  • Per-tool allow-from-public-chat toggle
  • Execution logs for accountability

Enterprise roadmap
